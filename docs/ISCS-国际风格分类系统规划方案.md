## 一、定义与范畴

### 1.1 核心定义

**国际风格分类系统(ISCS)** 是一套基于「文化地域 × 设计哲学 × 视觉语法」三维度的风格分类与生成框架。

其目标不是模仿某个国家的视觉外观,而是**翻译其设计思维与文化礼仪**,使同一份内容在不同文化语境中产生同等的说服力与信任感。

### 1.2 适用范畴

| 维度 | 说明 | 示例 |
|------|------|------|
| **地域文化** | 基于地理与文明圈的设计传统 | 日式、德式、美式、北欧、中东、中式 |
| **设计哲学** | 基于艺术运动与设计学派的方法论 | 包豪斯、瑞士国际主义、极简主义、和风 |
| **视觉语法** | 可量化执行的设计参数集 | 留白率、配色策略、网格系统、叙事结构 |

### 1.3 不包含范畴

- ❌ 纯文字翻译(已有能力,不在本系统范畴)
- ❌ 国旗/国徽等政治符号的直接使用
- ❌ 宗教教义的内容解读
- ❌ 对文化做优劣评判的排名体系

## 二、分类体系层级结构

### 2.1 三层分类架构

```
L1 文化圈 (Cultural Sphere)
├── L2 国别/地域 (Region)
│   ├── L3 风格变体 (Variant)
│   │   ├── L3a 经典变体
│   │   ├── L3b 现代变体
│   │   └── L3c 融合变体(跨文化混合)
```

### 2.2 L1 — 文化圈(6 大圈层)

| 代码 | 文化圈 | 覆盖地域 | 核心设计哲学 |
|------|--------|----------|------------|
| `EAS` | 东亚圈 | 中、日、韩、越 | 含蓄、留白、意境优先 |
| `CEU` | 中欧圈 | 德、瑞士、奥 | 严谨、网格、功能主义 |
| `WUS` | 西美圈 | 美、加、英 | 直接、冲击、故事驱动 |
| `NOR` | 北欧圈 | 瑞典、丹、芬、挪 | 极简、平等、自然 |
| `MEA` | 中东圈 | 沙特、UAE、埃及 | 对称、几何、关系优先 |
| `LAT` | 拉美圈 | 巴西、墨、阿根廷 | 温暖、人际、色彩 |

### 2.3 L2 — 国别/地域(示例)

```
EAS
├── JP  日本
├── CN  中国
├── KR  韩国
└── VN  越南

CEU
├── DE  德国
├── CH  瑞士
└── AT  奥地利
```

### 2.4 L3 — 风格变体(每国 2-4 个变体)

以日本为例:

| 变体代码 | 名称 | 特征 |
|---------|------|------|
| `JP-MIN` | 日式极简 | 40%+ 留白,明体,墨白 |
| `JP-TRD` | 日式传统 | 竖排标题,和纸纹理,家纹 |
| `JP-MOD` | 日式现代 | 无印良品风,灰阶,克制 |
| `JP-FUS` | 日式融合 | 日式留白 + 西式网格 |

### 2.5 划分标准

每个 L3 变体必须满足以下**可量化标准**才可入库:

| 参数 | 量化要求 |
|------|---------|
| 留白率 | 明确区间(如 35%-45%) |
| 配色数量 | 限定数量(如 ≤3 色) |
| 主色色相 | HSL 区间(如 H 0-15°, S 0-20%, L 15-30%) |
| 字体类别 | 明确推荐字体族(如 明体/游明朝) |
| 网格列数 | 明确列数(如 12 列 / 6 列) |
| 叙事弧线 | 定义 4-6 段结构(如 间接引入→背景→主题→数据→留白邀请) |
| 禁忌清单 | 至少 3 条文化禁忌规则 |

## 三、风格元素提取与整理

### 3.1 风格元素的四层结构

```
风格变体 (L3)
├── A. 视觉层参数    ← 可直接传给图像生成 API
├── B. 叙事层模板    ← 影响 LLM 大纲生成的结构
├── C. 禁忌层规则    ← 生成时的安全守卫
└── D. 元数据层      ← 标签、描述、案例引用
```

### 3.2 A 层 — 视觉参数(结构化)

每个变体以 JSON Schema 存储,直接可被 Prompt 引用:

```json
{
  "code": "JP-MIN",
  "visual": {
    "whitespace_ratio": { "min": 0.35, "max": 0.45 },
    "color_palette": {
      "primary":   "#1A1A1A",
      "secondary": "#FFFFFF",
      "accent":    "#B22222",
      "max_colors": 3
    },
    "typography": {
      "title_font": "YuMincho / Hiragino Mincho",
      "body_font": "Hiragino Sans / Noto Sans JP",
      "title_weight": 400,
      "line_height": 1.9
    },
    "grid": {
      "columns": 6,
      "gutter_ratio": 0.15
    },
    "imagery": {
      "style": "photographic minimal",
      "filter": "low-saturation",
      "icon_style": "line-thin"
    },
    "layout": {
      "alignment": "left",
      "title_position": "top-left",
      "text_density": "low"
    }
  }
}
```

### 3.3 B 层 — 叙事模板

每个变体定义大纲生成的结构模板:

```json
{
  "code": "JP-MIN",
  "narrative": {
    "opening": "industry_context_indirect",
    "structure": ["背景", "行业现状", "主题引入", "核心数据", "留白邀请"],
    "argumentation": "authority_citation_indirect",
    "climax": "single_quiet_number",
    "closing": "inviting_whitespace",
    "cta_style": "subtle"
  }
}
```

### 3.4 C 层 — 禁忌规则

```json
{
  "code": "JP-MIN",
  "taboos": [
    { "type": "number", "value": [4, 9], "context": "page_number", "action": "skip" },
    { "type": "color", "value": "pure_white_title", "reason": "丧事联想", "action": "warn" },
    { "type": "gesture", "value": "left_hand_point", "action": "remove" },
    { "type": "imagery", "value": "cherry_blossom_cliche", "action": "de_prioritize" }
  ]
}
```

### 3.5 D 层 — 元数据

```json
{
  "code": "JP-MIN",
  "name_zh": "日式极简",
  "name_en": "Japanese Minimal",
  "description": "基于余白の美学的克制设计,强调负空间与间",
  "design_philosophy": "Less is more, silence speaks",
  "inspiration_sources": ["无印良品", "原研哉", "田中一光"],
  "applicable_scenarios": ["内部汇报", "高端品牌", "文化提案"],
  "difficulty": "advanced",
  "created_at": "2026-07-10",
  "last_updated": "2026-07-10",
  "version": "1.0"
}
```

### 3.6 提取流程

```
1. 案例采集      ← 30+ 真实样本(公开设计作品/用户上传)
       │
2. 参数标注      ← 人工标注留白率/配色/字体/叙事结构
       │
3. 聚类分析      ← 提取共性参数区间
       │
4. 规则入库      ← 生成结构化 JSON Schema
       │
5. 专家校验      ← 设计/文化顾问审核
       │
6. 生成测试      ← 实际生成 5 份样本验证可执行性
       │
7. 正式发布      ← 入库并开放使用
```

## 四、标签命名规范与应用

### 4.1 命名规则

**格式**:`<L1>-<L2>-<L3>-<序号>`

| 层级 | 格式 | 示例 |
|------|------|------|
| L1 文化圈 | 3 字母大写 | `EAS` |
| L2 国别 | 2 字母大写(ISO 3166) | `JP` |
| L3 变体 | 3 字母大写 | `MIN` |
| 序号 | 2 位数字 | `01` |

**完整标签示例**:`EAS-JP-MIN-01`

### 4.2 应用规则

#### 4.2.1 生成时的标签使用

```
用户选择风格 → 解析标签 → 提取四层参数 → 注入 Prompt
```

- 视觉参数 → 拼入图像生成 Prompt
- 叙事模板 → 拼入大纲生成 Prompt
- 禁忌规则 → 生成后扫描拦截
- 元数据 → UI 展示

#### 4.2.2 多文化版本生成

当用户选择"多国版本"时,系统基于同一内容生成多个标签的产出:

```
输入: 大纲 + 目标标签列表 [EAS-JP-MIN-01, CEU-DE-BAU-01, WUS-US-DIR-01]
      ↓
并行生成 3 份同源不同体的 PPT
```

#### 4.2.3 文化融合标签

允许跨文化融合,命名规则:`<源A>+<源B>`

| 融合标签 | 说明 |
|---------|------|
| `EAS-JP-MIN + CEU-DE-BAU` | 日式留白 + 德式网格 |
| `NOR-SE-MIN + WUS-US-DIR` | 北欧极简 + 美式直接 |

融合时系统合并两套参数,冲突项(如留白率)取中值或让用户选择。

#### 4.2.4 用户自定义标签

允许高级用户基于现有变体创建自定义标签:

- 基础标签 + 用户覆写参数
- 命名规则:`<基础标签>-USR-<用户ID>`
- 例:`EAS-JP-MIN-01-USR-42`(用户 42 基于日式极简的自定义版本)

## 五、案例收集与展示策略

### 5.1 案例来源

| 来源 | 占比 | 说明 |
|------|------|------|
| 公开设计作品 | 40% | Behance/Dribbble/设计年鉴中标注文化风格的作品 |
| 历史经典案例 | 20% | 包豪斯作品、瑞士国际主义海报等 |
| 用户匿名上传 | 25% | 用户上传的优秀 PPT,脱敏后入库 |
| AI 生成样本 | 15% | 系统自身生成的验证样本 |

### 5.2 案例标注规范

每个案例必须标注:

```json
{
  "case_id": "CASE-2026-0001",
  "style_tag": "EAS-JP-MIN-01",
  "title": "无印良品 2024 品牌报告",
  "source": "public",
  "metrics": {
    "whitespace_ratio": 0.42,
    "color_count": 3,
    "page_count": 12
  },
  "visual_analysis": "base64_thumbnail_or_url",
  "tags": ["留白", "明体", "克制", "高端"],
  "verified_by": "design_expert",
  "verified_at": "2026-07-10"
}
  }
}
```

### 5.3 展示策略

#### 策略一:风格画廊(Style Gallery)

```
/g styles 路由
├── 按文化圈分组展示
├── 每个变体展示 3 个代表案例
├── 鼠标悬停查看参数详情
└── 点击案例 → "用此风格生成"
```

#### 策略二:对比视图(Compare View)

```
同一内容 × 3 种文化 = 3 列并排展示
├── 左:日式极简版
├── 中:德式严谨版
└── 右:美式直接版
      ↓
用户直观感受同一信息的不同文化表达
```

#### 策略三:文化光谱滑块

```
含蓄 ◄━━━━●━━━━━━━━━━━○━━━━━━━► 直接
  日式                    德式        美式

拖动滑块 → 实时预览该文化位置的视觉变化
```

#### 策略四:案例学习路径

```
/g learn 路由
├── 每个文化圈配套 3 节微课
│   ├── 视觉语法基础(10 分钟)
│   ├── 叙事结构解析(15 分钟)
│   └── 文化禁忌手册(5 分钟)

```

## 六、维护与更新机制

### 6.1 更新频率

| 层级 | 更新频率 | 触发条件 |
|------|---------|---------|
| L1 文化圈 | 年度 | 文明圈重大变迁时 |
| L2 国别 | 半年 | 新增国家支持 |
| L3 变体 | 季度 | 新增风格变体 |
| 案例库 | 月度 | 新案例入库 |
| 禁忌规则 | 月度 | 用户反馈 + 时事变化 |

### 6.2 用户反馈循环

```
用户生成 → 不满意 → 提交反馈("这个日式不够正宗")
      ↓
反馈进入队列 → 月度评审
      ↓
设计顾问审核 → 修订参数 → v1.1 版本
      ↓
A/B 测试 → 新旧版本对比 → 采纳更优者
```

### 6.3 版本管理

每个变体遵循语义化版本:

| 版本号 | 变更类型 | 示例 |
|--------|---------|------|
| Major (1.0 → 2.0) | 叙事结构重构 | 日式开场从"背景"改为"悬念" |
| Minor (1.0 → 1.1) | 视觉参数调整 | 留白率从 35%-45% 调整为 38%-48% |
| Patch (1.0.0 → 1.0.1) | 禁忌规则补充 | 新增"避免数字 42" |

### 6.4 质量守卫

- **自动校验**:每次参数修改后,自动生成 5 份样本,由 AI 评分(风格一致性 / 美观度 / 禁忌遵守)
- **人工抽检**:每月随机抽 10% 变体由设计顾问复核
- **用户评分**:生成后 1-5 星评分,低于 3 星的变体进入修订队列

## 七、文化差异与包容性设计

### 7.1 文化敏感度原则

| 原则 | 说明 | 落地方式 |
|------|------|---------|
| **去刻板印象** | 不把复杂文化简化为单一标签 | 每国提供 2-4 个变体,展现多样性 |
| **非评判性** | 不评价文化优劣 | 所有变体平等展示,无"最佳"标注 |
| **禁忌尊重** | 主动规避文化禁忌 | C 层禁忌规则强制执行 |
| **融合可能** | 鼓励跨文化混血 | 融合标签机制 |
| **持续学习** | 文化是活的,不是标本 | 季度更新 + 用户反馈循环 |

### 7.2 文化包容性检查清单

每个变体入库前必须通过以下检查:

- [ ] 是否包含该文化的多元面貌(非单一刻板)?
- [ ] 禁忌规则是否经过该文化背景顾问审核?
- [ ] 是否避免使用可能冒犯的符号/色彩/手势?
- [ ] 案例是否来自多元来源(非仅西方视角)?
- [ ] 描述文案是否中性、无优越感?

### 7.3 国际通用性保障

- **参数化而非模板化**:系统输出的是参数(留白率/配色),而非固定模板 — 适配任何内容
- **语言无关**:视觉语法不依赖特定语言,中/英/日/阿语均可应用
- **场景适配**:每个变体标注适用场景(汇报/提案/教学/营销),避免误用
- **无地域锁定**:用户在任何地区都可使用任何文化风格,不做地域限制

## 八、实施步骤与时间节点

### Phase 3:前端集成与测试(第 11-14 周)

| 周次 | 任务 | 交付物 |
|------|------|--------|

| W11-12 | 对比视图 + 文化光谱滑块 | `CompareView.vue` |

| W14 | 全流程测试 + Bug 修复 | 测试报告 |

**里程碑**:用户可在 UI 上选择文化风格并生成多版本 PPT。

### Phase 4:扩展与上线(第 15-18 周)

| 周次 | 任务 | 交付物 |
|------|------|--------|
| W15 | 扩展至 6 文化圈全覆盖(30+ 变体) | 完整风格库 |

**里程碑**:ISCS v1.0 正式上线,覆盖 6 大文化圈 30+ 风格变体。

interface StyleVariant {
  code: string;                    // EAS-JP-MIN-01
  name_zh: string;
  name_en: string;
  description: string;
  design_philosophy: string;

  visual: {
    whitespace_ratio: { min: number; max: number };
    color_palette: {
      primary: string;
      secondary: string;
      accent: string;
      max_colors: number;
    };
    typography: {
      title_font: string;
      body_font: string;
      title_weight: number;
      line_height: number;
    };
    grid: { columns: number; gutter_ratio: number };
    imagery: {
      style: string;
      filter: string;
      icon_style: string;
    };
    layout: {
      alignment: 'left' | 'center' | 'right';
      title_position: string;
      text_density: 'low' | 'medium' | 'high';
    };
  };

  narrative: {
    opening: string;
    structure: string[];
    argumentation: string;
    climax: string;
    closing: string;
    cta_style: string;
  };

  taboos: TabooRule[];
  metadata: StyleMetadata;
}

interface TabooRule {
  type: 'number' | 'color' | 'gesture' | 'imagery' | 'text';
  value: string | number[];
  context?: string;
  reason?: string;
  action: 'skip' | 'warn' | 'remove' | 'de_prioritize';
}
